Vision-based person identification struggles under real conditions:
\begin{itemize}
  \setlength\itemsep{1.62mm}
  \item \textbf{Low illumination} degraded visual features
  \item \textbf{Adverse weather} fog and rain reduce reliability

\end{itemize}

maintaining robustness without capturing identifiable imagery.

% ============================================================
% SECTION 2 -- CHALLENGES IN RADAR-BASED PID
% ============================================================
\block{\textcolor{white}{\large \ding{203}\ \ Challenges in Radar-Based PID}}{
\begin{minipage}{\linewidth}
\small
\posterfig{\linewidth}{tikz-radar-lib/problem_panel_strip.pdf}{Radar point clouds are sparse, noisy, and contaminated by multipath ghosts}
Radar-based person identification offers privacy and robustness, but faces four interconnected challenges:
\end{minipage}
\vspace{3mm}
\begin{minipage}{0.48\linewidth}
\textbf{Challenge 1: Poor Point Cloud Quality}\\[1mm]
Conventional angle-of-arrival processing yields sparse, noisy point clouds with limited angular resolution, motion-induced defocusing, and multipath ghost artifacts — especially in indoor environments with furniture, walls, and glass.
\end{minipage}
\hfill
\begin{minipage}{0.48\linewidth}
\textbf{Challenge 2: Rigid-Target Bias in InISAR}\\[1mm]
\vspace{1mm}
While Interferometric ISAR improves angular resolution, existing works validate it only on large \emph{rigid} targets (ships, aircraft) — \textbf{not} on small, \emph{articulated} human targets with low-RCS body segments.
\end{minipage}
\vspace{3mm}
\begin{minipage}{0.48\linewidth}
\textbf{Challenge 3: Multipath Cannot Be Ignored}\\[1mm]
DBSCAN alone cannot distinguish true targets from multipath ghosts. Ghosts overlap in range-Doppler and form clusters in 3D that mimic real objects — requiring temporal consistency for rejection.
\end{minipage}
\hfill
\begin{minipage}{0.48\linewidth}
\textbf{Challenge 4: Data Scarcity}\\[1mm]
Existing deep learning methods need $>$100 sequences/person (hours of recording). PointNet ignores local geometry; PCT handles spatial structure but targets \emph{static} objects, not temporally evolving radar sequences.
\end{minipage}
\vspace{3mm}
\begin{minipage}{\linewidth}
\centering
\colorbox{radarOrange!15}{\parbox{0.95\linewidth}{\centering\textbf{Gap:}
We bridge these gap with tracking-integrated InISAR and pre-trained few-shot meta-learning.}}
\end{minipage}
}

% ============================================================
% SECTION 4 -- InISAR PIPELINE: 3D RECONSTRUCTION
% ============================================================
\block{\textcolor{white}{\large \ding{206}\ \ InISAR Pipeline: 3D Reconstruction}}{
\small

% ---- Pipeline Figure ----
\posterfig{\linewidth}{tikz-radar-lib/inisar_pipeline_strip_landscape.pdf}{Six-stage 3D reconstruction pipeline}

\vspace{3mm}

% ---- Detailed Explanation in 2-Column Format ----
\begin{minipage}{0.48\linewidth}
\vspace{0mm}
\textbf{Step 1: Range Processing}
\begin{itemize}
    \setlength\itemsep{0.5mm}
    \item FMCW radar transmits linear frequency-modulated chirps:
\end{itemize}
\begin{align*}
    \Delta r = \frac{c}{2B} \quad \text{(range resolution)}
\end{align*}
\begin{itemize}
    \setlength\itemsep{0.5mm}
    \item Output: Range-compressed signal $S_{R,m}(f,t)$.
\end{itemize}
\vspace{1mm}
\textbf{Step 2: ISAR Motion Compensation (ICBA)}
\begin{itemize}
    \setlength\itemsep{0.5mm}
    \item Target motion during CPI smears energy across cells, defocusing the image.
    \item Translational motion model:
\end{itemize}
\begin{align*}
    R_0(t) = r_0 + \beta t + \gamma t^2
\end{align*}
\begin{itemize}
    \setlength\itemsep{0.5mm}
    \item ICBA maximizes image contrast to estimate $[\hat{\beta}, \hat{\gamma}]$:
\end{itemize}
\begin{align*}
    [\hat{\beta}, \hat{\gamma}] = \arg\max \left\{ \frac{\sqrt{\frac{1}{N}\sum(I_i - \bar{I})^2}}{\bar{I}} \right\}
\end{align*}
\vspace{1mm}
\textbf{Step 3: Doppler Processing}
\begin{itemize}
    \setlength\itemsep{0.5mm}
    \item Output: Range-Doppler map with focused target signatures.
\end{itemize}
\end{minipage}
\hfill
\begin{minipage}{0.48\linewidth}
\vspace{0mm}
\textbf{Step 4: CFAR Segmentation}
\begin{itemize}
    \setlength\itemsep{0.5mm}
    \item SOCA-CFAR ($N_T=6$, $P_{FA}=10^{-10}$) segments the target cells from noise/clutter.
\end{itemize}
\vspace{1mm}
\textbf{Step 5: Interferometry}
\begin{itemize}
    \setlength\itemsep{0.5mm}
    \item Uses phase difference between top and bottom antennas:
\end{itemize}
\begin{align*}
    I_1 \cdot I_2^* = |I_1||I_2| \exp\left(j\frac{2\pi}{\lambda}\Delta R\right)
\end{align*}
\begin{align*}
    \phi = \frac{2\pi}{\lambda} L\frac{h}{R}, \quad h = \frac{\lambda \phi R}{2\pi L}
\end{align*}
\begin{itemize}
    \setlength\itemsep{0.5mm}
    \item 3D coordinates:
\end{itemize}
\begin{align*}
    x = R\cos\theta_{el}\sin\theta_{az}
\end{align*}
\begin{align*}
    \quad y = R\cos\theta_{el}\cos\theta_{az}
\end{align*}
\begin{align*}
    z = R\sin\theta_{el}
\end{align*}
\vspace{1mm}
\textbf{Step 6: EKF-JPDA Tracking}
\begin{itemize}
    \setlength\itemsep{0.5mm}
    \item Tracks scatterers in 3D spatial domain (not Range-Doppler).
    \item \textbf{Why 3D?} Ghosts overlap in RD but separate in spatial domain.
    \item JPDA handles data association; EKF filters state with covariance.
    \item Output: Clean 3D point cloud with ghosts removed.
\end{itemize}
\end{minipage}

\vspace{2mm}

\begin{minipage}{0.48\linewidth}
\centering
\includegraphics[width=\linewidth]{fig_radar_vs_camera_p3_0053.png}
\small\textcolor{radarGrey}{(a) Before removal: ghost targets remain}
\end{minipage}
\hfill
\begin{minipage}{0.48\linewidth}
\includegraphics[width=\linewidth]{fig_radar_vs_camera_p5_0065.png}
\small\textcolor{radarGrey}{(b) After removal: ghosts rejected}
\end{minipage}

% ---- Key Takeaway ----
\vspace{2mm}
\centering
\colorbox{radarOrange!15}{\parbox{0.95\linewidth}{\centering
\textbf{Key Insight:} Tracking in 3D spatial domain (not Range-Doppler) is the key to rejecting multipath ghosts — ghosts overlap in RD but form separable trajectories in 3D.
}}
}

\block{\textcolor{white}{\large \ding{211}\ \ Pre-trained MAML}}{
    \small
    \textbf{Two-Phase Algorithm:}
    \begin{enumerate}
        \setlength\itemsep{2pt}
        \item \textbf{Phase 1:} Supervised training on representative task (3 subjects) $\to$ $\theta_0^{PT}$
        \item \textbf{Phase 2:} Meta-train MAML with $\theta_0^{PT}$ initialization
    \end{enumerate}
    \vspace{3mm}
    \begin{align*}
        \text{Inner Loop: } & \theta_i' = \theta - \alpha \nabla \mathcal{L}_{\mathcal{T}_i} \\
        \text{Outer Loop: } & \theta \leftarrow \theta - \beta \nabla \sum_i \mathcal{L}_{\mathcal{T}_i}
    \end{align*}
    \small
    \textbf{Key Advantage:} $\theta_0^{PT}$ is closer to optimal $\theta^*$ than $\theta_0^{rand}$,
    reducing meta-training iterations.
}

\textit{Path forward:} multimodal fusion where privacy allows\vspace{1mm}}}

\hfill
\begin{minipage}{0.19\linewidth}
\centering
\colorbox{radarBlue!15}{\parbox{0.95\linewidth}{\vspace{1mm}

\textit{Path forward:} community dataset + HPC access\vspace{1mm}}}
\end{minipage}
\hfill
\begin{minipage}{0.19\linewidth}
\centering
\colorbox{radarBlue!15}{\parbox{0.95\linewidth}{\vspace{1mm}

\textit{Path forward:} faster tracking algorithms\vspace{1mm}}}

\hfill
\begin{minipage}{0.19\linewidth}
\centering
\colorbox{radarBlue!15}{\parbox{0.95\linewidth}{\vspace{1mm}
